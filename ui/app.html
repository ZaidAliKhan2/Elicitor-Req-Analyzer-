About Elicitor

Elicitor is an intelligent requirements analyzer that helps you classify and validate software requirements for your projects. It uses machine learning to determine if requirements are in scope and categorizes them as functional or non-functional.

Requirement Classifications

FR - Functional Requirements

Requirements that describe what the system should do. These define specific behaviors, functions, or features.

  • User authentication and login functionality
  • Product catalog and search features
  • Shopping cart and checkout process
  • Payment processing integration
  • Order management and tracking

NFR - Non-Functional Requirements

Requirements that describe how the system should perform. These define quality attributes and constraints.

NFR Subcategories:

  • Performance: Speed, response time, throughput (e.g., "Page load time under 2 seconds")
  • Security: Authentication, encryption, data protection (e.g., "All data must be encrypted")
  • Usability: User experience, accessibility, ease of use (e.g., "Interface should be mobile-friendly")
  • Reliability: Uptime, availability, fault tolerance (e.g., "99.9% uptime guarantee")
  • Scalability: Growth capacity, concurrent users (e.g., "Support 10,000 concurrent users")
  • Maintainability: Code quality, modularity, documentation (e.g., "Code must be well-documented")
  • Portability: Platform independence, cross-compatibility, adaptability (e.g., "System must run on Windows, Mac, and Linux")
  • Legal: Compliance, regulations, licensing, data privacy laws (e.g., "Must comply with GDPR regulations")

How It Works

1. Set Your Project: Describe your project domain to establish scope boundaries.

2. Submit Requirements: Enter requirements one at a time for analysis.

3. Review Analysis: See scope status, confidence score, and classification results.

Understanding Results

  • Scope Status: Whether the requirement fits within your project domain
  • Confidence Score: How certain the model is about its classification (0-100%)
  • Classification: Whether it's FR, NFR, or other type
  • Subcategory: For NFRs, the specific quality attribute category
